# 自学习技能

犯错后自动提取经验、固化规则、避免重犯。

## 核心原则

**一句话：犯错不可怕，重犯才可怕。**

1. **每次被纠正 → 必须更新经验库**（不等待用户指令）
2. **经验必须具体可执行**（抽象原则 + 具体检查清单）
3. **做事前查阅经验库**（避免重犯已知错误）
4. **经验分两层存储**（SKILL.md 存精简规则，references/ 存详细案例）

## 工作流程

### 阶段1：捕获错误（实时）

当出现以下信号时，立即进入经验提取流程：
- 用户说"你做错了"、"不对"、"应该用xxx方法"
- 命令执行失败但被手动修复
- 用户说"注意一下"、"下次要xxx"
- 自身发现使用了错误的方法/工具

### 阶段2：提取经验

每个经验记录包含：

```
错误：具体发生了什么
原因：为什么会犯错（思维偏差/知识缺失/流程遗漏）
修正：正确的做法是什么
规则：抽象为可执行原则（1句话）
场景：在什么情况下需要应用此规则
```

### 阶段3：固化到经验库

**写入 references/experiences.md**，追加格式：
```markdown
## [YYYY-MM-DD] 经验标题
- **错误**：xxx
- **原因**：xxx  
- **规则**：xxx（可执行原则）
- **场景**：当 xxx 时，必须 xxx
```

### 阶段4：更新 SKILL.md 精简规则

**仅当规则具有通用性时**，更新本文件 [精简规则库](#精简规则库) 章节。
- 每条规则不超过1行
- 去重，避免重复条目
- 优先保留更精确的表述

## 精简规则库

> 以下是从经验中提取的核心规则，做事前必须检查。

1. **创建Obsidian笔记必须包含YAML frontmatter** — 用 `---` 包围，包含 title、tags 等属性
2. **使用专用技能/工具后必须验证结果** — 不能假设成功
3. **优先使用专用技能而非通用方法** — 有Obsidian CLI就不用文件写入
4. **执行外部操作（保存/发送/删除）前先确认目标位置正确**
5. **非主通道（微信/webchat）生成的Markdown内容不可信** — 必须读取文件验证格式，特别是代码块（反引号可能被转义或插入控制字符）
6. **Windows PowerShell中用分号`;`连接命令，不用`&&`** — PowerShell 5.1不支持bash风格的`&&`
7. **写PowerShell脚本(.ps1)必须用qclaw-text-file技能** — 确保正确的utf-8-sig编码和BOM

## 预防检查清单

> 执行重要操作前，快速过一遍：

- [ ] 是否有对应的专用技能？→ 优先使用
- [ ] 操作完成后是否需要验证？→ 必须验证
- [ ] 经验库中是否有相关经验？→ 先查阅
- [ ] 操作是否可逆？→ 不可逆操作需额外确认

## 维护规则

- 经验记录累超过50条时，做一次归纳合并
- 过时的经验标注 `[已过时]` 但保留（防止反复犯错）
- 每周回顾一次，提取新的通用规则到精简规则库
